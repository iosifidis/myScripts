# Universal Linux Backup & Restore Scripts

Ένα σύνολο από scripts για τη δημιουργία αντιγράφων ασφαλείας (backup) και την επαναφορά των ρυθμίσεων και των εγκατεστημένων προγραμμάτων σε διαφορετικές διανομές Linux. Ο στόχος είναι η γρήγορη και αυτοματοποιημένη ρύθμιση ενός νέου συστήματος ώστε να μοιάζει με το παλιό, χωρίς την ανάγκη χειροκίνητων ενεργειών.

## Βασικά Χαρακτηριστικά

-   **Cross-Platform:** Σχεδιασμένο για να λειτουργεί σε οικογένειες διανομών όπως **Fedora**, **Debian/Ubuntu** και **openSUSE**.
-   **Πλήρης Υποστήριξη Πακέτων:** Υποστηρίζει native πακέτα (`dnf`, `apt`, `zypper`), **Flatpaks** και **Snaps**.
-   **Αντίγραφα Ασφαλείας Ρυθμίσεων:** Δημιουργεί backup για:
    -   Σημαντικά αρχεία ρυθμίσεων (dotfiles).
    -   Ρυθμίσεις-κλειδιά για εφαρμογές όπως το VS Code και το Google Chrome.
    -   Αποθηκευμένους κωδικούς Wi-Fi.
    -   Προσωπικά fonts, themes και icons.
-   **Ανθεκτική Επαναφορά με Ansible:** Η διαδικασία επαναφοράς γίνεται μέσω ενός Ansible playbook, το οποίο είναι ιδιοδύναμο (idempotent), δηλωτικό και αξιόπιστο.

## Δομή Αρχείων

-   `universal_backup.sh`: Το εκτελέσιμο script που τρέχεις στο παλιό σου σύστημα για να δημιουργήσεις το backup.
-   `universal_restore.yml`: Το Ansible playbook που τρέχεις στο νέο σου σύστημα για να επαναφέρεις τα πάντα.

## Οδηγίες Χρήσης

Η διαδικασία αποτελείται από δύο μέρη: τη δημιουργία του backup και την επαναφορά του.

### Μέρος 1: Δημιουργία του Backup

Τρέξε τα παρακάτω βήματα στο **παλιό σου σύστημα**, αυτό που θέλεις να αποθηκεύσεις.

1.  **Κάνε το script εκτελέσιμο:**
    ```bash
    chmod +x universal_backup.sh
    ```

2.  **Τρέξε το script:**
    ```bash
    ./universal_backup.sh
    ```
    Το script θα σου ζητήσει τον κωδικό σου (`sudo password`) για να αποθηκεύσει αρχεία συστήματος όπως τα αποθετήρια και τους κωδικούς Wi-Fi.

3.  **Τι συμβαίνει:** Το script θα δημιουργήσει έναν νέο φάκελο `~/linux-backup`. Αυτός ο φάκελος περιέχει:
    -   Λίστες με όλα τα εγκατεστημένα σου προγράμματα (`dnf_packages.list`, `apt_packages.list`, `flatpak_packages.list`, `snap_packages.list`).
    -   Έναν φάκελο `home_settings` με τα επιλεγμένα σου dotfiles.
    -   Έναν φάκελο `repos` με τις ρυθμίσεις των αποθετηρίων σου.
    -   Έναν φάκελο `network_connections` με τους κωδικούς του Wi-Fi.

4.  **Αποθήκευση του Backup:**
    **Αυτό είναι το πιο σημαντικό βήμα!** Αντέγραψε **ολόκληρο τον φάκελο `linux-backup`** σε έναν εξωτερικό σκληρό δίσκο, USB stick ή σε κάποια υπηρεσία cloud.

### Μέρος 2: Επαναφορά σε Νέο Σύστημα

Αφού έχεις κάνει format και έχεις εγκαταστήσει μια νέα διανομή Linux, ακολούθησε τα παρακάτω βήματα.

1.  **Προαπαιτούμενα:** Πριν τρέξεις το playbook, πρέπει να εγκαταστήσεις το Ansible και τις απαραίτητες συλλογές.
    
    *   Για **Fedora**:
        ```bash
        sudo dnf install ansible -y
        ```
    *   Για **Debian/Ubuntu**:
        ```bash
        sudo apt update
        sudo apt install ansible -y
        ```
    *   Για **openSUSE**:
        ```bash
        sudo zypper install ansible -y
        ```
        
    *   **Εγκατάσταση της Ansible collection** (απαραίτητο για το Flatpak module):
        ```bash
        ansible-galaxy collection install community.general
        ```

2.  **Μεταφορά του Backup:** Αντέγραψε τον φάκελο `linux-backup` από τον εξωτερικό σου δίσκο στον home φάκελο του νέου συστήματος (`~/`).

3.  **Εκτέλεση του Playbook:** Άνοιξε ένα terminal στον φάκελο όπου έχεις το αρχείο `universal_restore.yml` και τρέξε την εντολή:
    ```bash
    ansible-playbook universal_restore.yml --ask-become-pass
    ```
    Το Ansible θα σου ζητήσει τον κωδικό σου (`BECOME password:`) για να εκτελέσει τις εργασίες που απαιτούν δικαιώματα διαχειριστή.

4.  **Ολοκλήρωση:** Το playbook θα εγκαταστήσει όλα τα προγράμματα και θα επαναφέρει τις ρυθμίσεις. Μόλις τελειώσει, κάνε μια **επανεκκίνηση** για να βεβαιωθείς ότι όλες οι αλλαγές έχουν εφαρμοστεί σωστά.

## Προσαρμογή

Μπορείς εύκολα να προσαρμόσεις τα scripts στις δικές σου ανάγκες.

### Προσθήκη Dotfiles στο Backup

Άνοιξε το αρχείο `universal_backup.sh` και βρες το παρακάτω τμήμα:

```bash
# ...
cat <<EOF > "$BACKUP_DIR/dotfiles.list"
.bashrc
.bash_profile
.gitconfig
.ssh
.fonts
.icons
.themes
.local/share/keyrings
EOF
# ...
```
Απλά πρόσθεσε μια νέα γραμμή με το όνομα του αρχείου ή του φακέλου (που ξεκινά με τελεία) που θέλεις να συμπεριλάβεις στο backup.

### Προσθήκη Συγκεκριμένων Ρυθμίσεων

Αν θέλεις να προσθέσεις τις ρυθμίσεις από μια άλλη εφαρμογή, μπορείς να ακολουθήσεις το παράδειγμα των Chrome/Code και να προσθέσεις μια εντολή `cp` στο τέλος του script, πάντα ελέγχοντας αν το αρχείο υπάρχει:

```bash
if [ -f "$HOME/.config/MιαΑλληApp/settings.conf" ]; then
    cp "$HOME/.config/MιαΑλληApp/settings.conf" "$BACKUP_DIR/home_settings/.config/MιαΑλληApp/"
fi
```

## Περιορισμοί και Σημειώσεις

-   **Επαναφορά μεταξύ διαφορετικών οικογενειών διανομών:** Το σύστημα είναι σχεδιασμένο κυρίως για επαναφορά σε διανομή της **ίδιας οικογένειας** (π.χ. από Fedora σε Fedora, ή από Ubuntu σε Mint). Η προσπάθεια επαναφοράς ενός backup από Fedora σε Ubuntu **δεν θα λειτουργήσει** για τα native πακέτα (`dnf_packages.list` vs `apt_packages.list`), καθώς τα ονόματα των πακέτων διαφέρουν.
-   **PPA GPG Keys (Ubuntu):** Το script αποθηκεύει τα αρχεία των PPA, αλλά όχι τα κλειδιά GPG που απαιτούνται για την αυθεντικοποίησή τους. Αυτά μπορεί να χρειαστεί να προστεθούν χειροκίνητα.
-   **Ασφάλεια:** Το backup περιέχει ευαίσθητες πληροφορίες, όπως τα SSH keys και τους κωδικούς του Wi-Fi. **Φύλαξε τον φάκελο `linux-backup` σε ασφαλές μέρος!**
